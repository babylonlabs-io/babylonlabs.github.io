## Overview

`babylond` CLI commands reference

- [`add-genesis-account`](../add_genesis_account): Add a genesis account to `genesis.json`

- [`collect-gentxs`](../collect_gentxs) : Collect genesis transactions and output a genesis.json file

- [`comet`](../comet) : CometBFT subcommands

- [`config`](../config) : Utilities for managing application configuration

- [`create-bls-key`](../create_bls_key) : Create a pair of BLS keys that are used to send BLS signatures for checkpointing.

- [`debug`](../debug) : Debugging utilities

- [`export`](../export) : Export blockchain state

- [`gen-helpers`](../gen_helpers) : Useful commands for creating the genesis state with Babylon-specific Bitcoin staking data

- [`gen-tx`](../gen_tx) : Generate a transaction

- [`generate-bls-pop`](../generate_bls_pop) : Generate a BLS keypair

- [`help`](../help) : Help about any command

- [`init`](../init) : Node initialization

- [`keys`](../keys) : Key management

- [`migrate`](../migrate) : Migrate from one protocol version to another

- [`module-hash-by-height`](../module_hash_by_height) : Get the module hash at a specific height

- [`module-sizes`](../module_sizes) : Get the module sizes

- [`prepare-genesis`](../prepare_genesis) : Prepare genesis.json file

- [`query`](../query) : Query the network

- [`rollback`](../rollback) : Perform state rollback

- [`start`](../start) : Run the full node application with CometBFT in or out of process.

- [`status`](../status) : Query node for status

- [`testnet`](../testnet) : Test the application

- [`tx`](../tx) : Send a transaction

- [`validate-genesis`](../validate_genesis) : Validate the genesis file

- [`version`](../version) : Print the version of the application
