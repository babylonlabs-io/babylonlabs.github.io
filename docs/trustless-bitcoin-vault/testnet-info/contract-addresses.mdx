# Contract addresses

This page lists the deployed contract addresses for the current Trustless Bitcoin Vault (TBV) Testnet release on **Sepolia** (chain ID `11155111`). **Audience:** developers, integrators, and security researchers. Depositors using the demo app do not need to interact with contracts directly.

For wallet and network setup, see [Setup](./setup). For the live values of protocol parameters (thresholds, timeouts), see [Protocol parameters](./protocol-parameters).

Addresses can change between Testnet releases. Always check this page before sending a transaction or interacting with a contract directly.

## Bitcoin side

The Bitcoin side has no protocol contracts: each vault corresponds to a single Taproot UTXO created at peg-in. The vault's spending paths are committed in the Taproot script at creation. See [Protocol architecture](../technical-details/protocol-architecture) for the script structure.

## Per-user contracts

The AaveAdapterPositionProxy is deployed per depositor, on first interaction with the adapter. There is no single global address; the proxy for a given depositor can be looked up through the AaveAdapterLens or by inspecting the deploying transaction.

| Contract | Purpose | How to find |
| --- | --- | --- |
| `AaveAdapterPositionProxy` | Per-depositor isolated lending position; deployed deterministically on first deposit | Query `AaveAdapterLens.getPosition(userAddress)` or inspect the user's first interaction transaction |

## Read-only query helpers

Functions developers can call to read protocol state without sending a transaction:

* `BTCVaultRegistry.getPegInFee(vaultProvider)` — current peg-in fee for a chosen VP
* `BTCVaultRegistry.getVaultProviderCommission(vpAddress)` — VP's commission in BPS
* `BTCVaultRegistry.getBtcVaultBasicInfo(vaultId)` — vault status and basic info
* `AaveAdapter.getPosition(depositorEOA)` — position summary
* `AaveAdapterPositionProxy.getUserAccountData()` — health factor and values
* `AaveAdapterPositionProxy.getReserveTotalDebt(reserveId)` — current debt per reserve
* `CapPolicy.getAvailableAppCapacity(appEntryPoint)` — remaining application cap
* `CapPolicy.getAvailableUserAppCapacity(appEntryPoint, user)` — remaining user cap
