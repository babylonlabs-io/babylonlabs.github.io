## Known limitations

The Testnet is a beta. Issues that may surface and how they are handled:

* **Contract changes between releases.** The team can redeploy contracts during Testnet. Always check [Contract addresses](../testnet-info/contract-addresses) before transacting.
* **Operator dependencies.** Activation requires the [Vault Provider and Vault Keepers](../technical-details/protocol-actors) to be online. Two distinct timeouts apply: `pegInAckTimeout` (~24h) for off-chain setup completion, and `pegInActivationTimeout` (~48h) for the depositor's secret reveal. If the Vault Provider is unreachable for redemption, the depositor's self-claim path (WOTS + claimer artifacts via a command-line tool the protocol ships) remains available for already-active vaults. See [Withdraw & redeem](../use-for-lending/withdraw-and-redeem).
* **Conservative caps.** Per-address (`perAddressCapBTC` = 0.4 BTC) and per-application (`totalCapBTC` = 10 BTC for the Aave application) caps are intentionally tight on Testnet. These limits will grow as the protocol matures.
* **Pause states.** The protocol can enter soft pause or full pause in response to incidents, by 3-of-5 Security Council quorum. Behaviour during pause is described in [Safety & trust assumptions](../start-here/safety-and-trust-assumptions).

## Path to mainnet

Mainnet deployment will be a distinct release process from Testnet:

* **Separate audit pass.** The production codebase will receive its own audit, scoped to mainnet contracts and parameters.
* **Expanded Security Council.** The Security Council quorum will move from team-controlled keys to an independent-signer setup.
* **Wider operator sets.** Multiple Vault Providers competing on commission, an expanded Vault Keeper set per application, and an opened-up Universal Challenger registry.
* **Bug bounty transition.** The bug bounty program will transition from Testnet best-effort scope to production scope with full reward tiers.
* **Production-scale caps.** Per-application and per-address BTC caps will be raised significantly, with values published when mainnet parameters are finalized.
